   ├─ 创建 EndTruncatingAccumulator（防 stdout 无限增长）

   └─ 返回 ExecResult

3. 结果后处理
   ├─ trackGitOperations（git 状态追踪）
   ├─ interpretCommandResult（语义退出码解释）
   ├─ resetCwdIfOutsideProject（子 agent 不允许改 cwd）
   ├─ SandboxManager.annotateStderrWithSandboxFailures
   └─ ShellError throw（非零退出码 + isError = true）

4. 输出处理
   ├─ stripEmptyLines
   ├─ extractClaudeCodeHints（零 token 侧通道）
   ├─ isImageOutput → resizeShellImageOutput
   ├─ >30K → persistedOutputPath（硬链接到 tool-results）
   └─ > 64MB → fsTruncate

5. Analytics
   └─ tengu_bash_tool_command_executed + code_indexing_tool_used

RemoteTriggerTool（3 文件/192行）允许外部系统通过 API 触发 Claude 执行动作—— GitHub webhook、CI/CD 事件、监控告警等可以作为触发源。触发器携带 payload（JSON）， 注入为 UserMessage 进入 Query Loop。在 Kairos 模式下，这使 Claude 能够 响应外部事件而不仅仅是用户输入。

🔑 一句话总结：工具深剖系列完整覆盖了 24 个核心工具。BriefTool / ScheduleCronTool / RemoteTriggerTool 构成了 Kairos（Proactive Agent）体系的工具三角——BriefTool 是「嘴」（输出通道），CronTool 是「时钟」（调度器），RemoteTriggerTool 是「耳朵」（外部事件感知），三者协同让 AI 从被动响应者进化为主动执行者。
