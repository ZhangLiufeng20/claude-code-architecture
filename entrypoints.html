30.1 CLI 引导层 (cli.tsx)

cli.tsx 是 Claude Code 的 真正入口文件。它的设计核心是 Fast-Path 架构——在 main() 函数中，通过一系列 if 分支实现 14+ 种命令路径的懒加载路由：

flowchart TB ENTRY["void main()"] --> ARGS["解析 process.argv"] ARGS --> V{"--version / -v?"} V -->|yes| PRINT["console.log(MACRO.VERSION)\n零 import 返回"] V -->|no| PROFILER["加载 startupProfiler"] PROFILER --> DUMP{"--dump-system-prompt?"} DUMP -->|yes| SYSPROMPT["输出渲染后的 System Prompt"] DUMP -->|no| CHROME{"--claude-in-chrome-mcp?"} CHROME -->|yes| MCPCHROME["Chrome MCP Server"] CHROME -->|no| DAEMON{"--daemon-worker?"} DAEMON -->|yes| WORKER["Daemon Worker"] DAEMON -->|no| BRIDGE{"remote-control/rc/bridge?"} BRIDGE -->|yes| BRIDGEMAIN["Bridge Main + Auth + Policy"] BRIDGE -->|no| BG{"ps/logs/attach/kill/--bg?"} BG -->|yes| BGHANDLER["bg.js 后台会话管理"] BG -->|no| TPL{"new/list/reply?"} TPL -->|yes| TEMPLATES["Template Jobs"] TPL -->|no| RUNNER{"environment-runner?"} RUNNER -->|yes| ENVRUNNER["BYOC 环境执行器"] RUNNER -->|no| FULL["加载 main.js\n完整 CLI 启动"]

顶层副作用（Top-Level Side Effects）

  • COREPACK_ENABLE_AUTO_PIN = '0'：修复 corepack 自动钉版本导致 yarnpkg 被写入 package.json 的 bug
  • NODE_OPTIONS += --max-old-space-size=8192：CCR（Cloud Container Runtime）环境下，将子进程堆限制设为 8GB（容器共 16GB）
  • ABLATION_BASELINE 环境变量注入：Harness-science L0 实验基线——一次性禁用 SIMPLE/THINKING/INTERLEAVED/COMPACT/AUTO_MEMORY/BACKGROUND_TASKS 6 个特性
⚡ 性能关键：所有 Fast-Path 均使用 await import(...) 动态导入——确保 --version 路径的冷启动为 亚毫秒级。 feature() 调用内联（非函数导入）以支持构建时 Dead Code Elimination (DCE)。

30.4 Agent SDK 类型系统 (agentSdkTypes.ts)

这是 Claude Code 暴露给外部开发者的 SDK 公共 API 类型定义。设计为 "类型桩" (Type Stubs)：所有函数仅包含 throw new Error('not implemented')，在构建时由真正的运行时实现替换。

函数稳定性功能

createSdkMcpServer()稳定创建进程内 MCP Server 实例

unstable_v2_resumeSession()@alphaV2 恢复已有会话
unstable_v2_prompt()@alphaV2 单次便捷 prompt
getSessionMessages()稳定读取 JSONL 转录记录

watchScheduledTasks()@internalDaemon Cron 调度器
connectRemoteControl()@internalDaemon WebSocket Bridge

类型架构分层

  • sdk/coreTypes.ts：由 Zod Schema 生成的可序列化类型（HOOK_EVENTS × 24、EXIT_REASONS × 6）
  • sdk/runtimeTypes.ts：不可序列化的运行时类型（回调、接口）
  • sdk/controlTypes.ts：SDK 控制协议（Bridge 子路径消费者）
  • sdk/toolTypes.ts：工具类型（@internal 直到 API 稳定）
  • sdk/settingsTypes.generated.ts：从 JSON Schema 生成的设置类型

30.5 Sandbox 类型定义 (sandboxTypes.ts)

Sandbox 配置的 单一真相源（Single Source of Truth），使用 Zod v4 Schema 定义，同时被 SDK 和 settings 验证层引用：

Schema核心字段设计意图
SandboxNetworkConfigSchemaallowedDomains · allowUnixSockets · httpProxyPort · socksProxyPort网络白名单 + MITM 代理配置
SandboxFilesystemConfigSchemaallowWrite · denyWrite · denyRead · allowRead文件系统读写权限（与 Edit/Read 权限规则合并）
SandboxSettingsSchemaenabled · failIfUnavailable · autoAllowBashIfSandboxed · enableWeakerNestedSandbox沙箱总开关 + 安全降级策略

关键安全设计

  • allowManagedDomainsOnly：当由 managed settings 设置时，仅尊重管理源的域名白名单，用户/项目/本地设置的域名被忽略
  • allowManagedReadPathsOnly：当由 managed settings 设置时，仅使用管理源的 allowRead 路径
  • failIfUnavailable：企业部署模式——沙箱不可用时直接 exit 而非降级运行
  • enableWeakerNetworkIsolation：macOS 专用——允许 trustd 服务访问以支持 Go CLI 工具的 TLS 证书验证（降低安全性）
  • enabledPlatforms：隐藏字段（.passthrough()），限制沙箱开启的平台（为 NVIDIA 企业部署设计）
  • excludedCommands：特定命令绕过沙箱列表
🔑 一句话总结：entrypoints/ 是 Claude Code 的 进程引导基础设施。cli.tsx（303行）通过 14+ 条 Fast-Path 路由实现极致懒加载——--version 零 import 返回，完整启动仅在默认路径触发；init.ts（341行）以 memoize 保证幂等性，精确编排 15+ 项初始化任务的并发拓扑（fire-and-forget 异步 + 同步阻塞约束）；mcp.ts 将全部工具暴暴露为 MCP Stdio 服务器；agentSdkTypes.ts 定义了 V1/V2 双版本 SDK 公共 API（10+ 函数桩）；sandboxTypes.ts 用 Zod v4 构建了网络/文件系统/总开关三层沙箱安全 Schema。
