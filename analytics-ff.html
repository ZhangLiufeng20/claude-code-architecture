Feature Flag 三种读取模式

API阻塞？场景一致性
_CACHED_MAY_BE_STALE❌ 同步启动关键路径、render 循环最终一致（磁盘缓存可能跨进程陈旧）
_BLOCKS_ON_INIT⚠️ 等 initkill switch、动态配置Init 后强一致
checkSecurityRestrictionGate⚠️ 等重初始化安全门控Auth 变更后强一致
checkGate_CACHED_OR_BLOCKING⚠️ 仅 false 时阻塞订阅权限（缓存 true → 快路径）false 时强一致

5 级优先级读取链

优先级来源说明
1环境变量覆盖CLAUDE_INTERNAL_FC_OVERRIDES（ant-only，eval harness 用）
2/config Override配置页面 Gates Tab（ant-only，运行时可改）

4磁盘缓存~/.claude.json 中的 cachedGrowthBookFeatures（跨进程存活）
5默认值调用方传入的 defaultValue

30.3 GrowthBook 生命周期

初始化流程

  • Trust 检查：未建立 Trust → 跳过 HTTP init，使用磁盘缓存
  • Auth 注入：getAuthHeaders() 注入到 apiHostRequestHeaders，客户端创建后不可变
  • SDK 补丁：API 返回 {value: ...} 但 SDK 期望 {defaultValue: ...}，processRemoteEvalPayload 做格式转换
  • 空 payload 防护：{features: {}} 不会清空缓存，防止 "flag 全黑"

Auth 变更重初始化

  • 登录/登出 → refreshGrowthBookAfterAuthChange()
  • 销毁旧客户端（apiHostRequestHeaders 不可更新），清空全部缓存
  • 设置 reinitializingPromise，安全门控 checkSecurityRestrictionGate 会 await 这个 Promise
  • 初始化完成 → refreshed.emit() → 订阅者（模型选择、权限等）重建

周期刷新

  • ant 用户：20 分钟一次（快速迭代门控）
  • 外部用户：6 小时一次（与 Statsig 对齐）
  • 使用 setInterval().unref()，不阻止进程自然退出
  • 刷新后 processRemoteEvalPayload → syncRemoteEvalToDisk → refreshed.emit()

30.4 实验曝光与 Statsig 迁移

曝光去重

  • loggedExposures: Set<string> — 每个 feature 在会话中最多触发一次曝光事件
  • pendingExposures: Set<string> — init 完成前访问的 feature 延迟记录
  • 曝光通过 logGrowthBookExperimentTo1P 写入 1P 事件系统

Statsig → GrowthBook 迁移层

  • checkStatsigFeatureGate_CACHED_MAY_BE_STALE：先查 GrowthBook 缓存，降级到 Statsig 缓存（cachedStatsigGates）
  • 安全门控 checkSecurityRestrictionGate 反过来——先查 Statsig 缓存（安全优先），再查 GrowthBook

🔑 架构洞察：GrowthBook 不仅仅是"特性开关"，它是 Claude Code 的 运行时行为控制平面。 从模型选择（tengu_ant_model_override）、安全限制（checkSecurityRestrictionGate） 到版本 kill switch（tengu_max_version_config），全部通过 GrowthBook 远端控制。 5 级读取优先级 + 3 种访问模式确保了从"启动速度"到"安全一致性"的全覆盖。 磁盘缓存跨进程存活，防止网络不可达时全部降级到默认值。
