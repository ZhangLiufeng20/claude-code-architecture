关键设计决策

  • 破解循环依赖：Permission 类型从 utils/permissions/ 抽取到 types/permissions.ts，实现文件可微嘉参引而不产生运行时依赖
  • 品牌类型 (Branded Types)：ids.ts 使用 TypeScript 品牌类型确保 SessionId/AgentId/TaskId 不可互换
  • generated 自动生成：从 Proto/OpenAPI schema 生成的类型只读、不可编辑，确保 API 契约一致性
🔑 一句话总结：types/ 是 Claude Code 的 类型基座。通过将核心接口（权限 442行/插件/Hooks/命令/日志）抽取到纯类型文件，避免循环依赖；Branded Types（SessionId/AgentId）给各 ID 加上类型安全屏障；generated/ 自动生成确保 API 契约一致。

33.3 DreamTask — 推测性执行引擎

DreamTask 是 Claude Code 最具创新性的任务类型——在用户 空闲时自动推测接下来可能需要执行的任务， 预先运行并缓存结果。当用户实际发出相应指令时，系统直接返回已缓存的结果，实现"零等待"体验。 这是 Claude Code 独有的"预测式 AI 编程"范式，类似于 CPU 的分支预测机制。

特性实现说明
触发条件用户 idle 超过阈值 + 有可推测的执行计划后台静默执行，不干扰前台
沙箱隔离独立工作目录 + 权限限制推测执行不能修改生产文件
结果缓存DreamResult → TaskRegistry 存储命中时直接返回，未命中时丢弃
资源控制低优先级 CPU + token 预算上限不影响前台任务性能

33.4 LocalShellTask — 后台 Shell 执行器

LocalShellTask（3 文件/5KB）是最常用的后台任务类型。通过 spawn 创建独立子进程， 支持 stdout/stderr 实时捕获、SIGTERM 优雅退出、tmux session 自动管理。

生命周期行为安全约束
创建fork 子进程 + stdin 管道 + 输出缓冲继承父进程 sandbox 限制
监控定期轮询 exit code + tail output输出截断防 OOM（MAX_OUTPUT_BYTES）
更新通过 stdin 发送消息/信号仅允许 UTF-8 文本输入
终止SIGTERM → 等待 5s → SIGKILL清理 tmux session + 临时文件

33.5 InProcessTeammateTask — 同进程协作

InProcessTeammateTask（2 文件）在 同一 Node 进程内运行 teammate agent， 通过共享内存通信（EventEmitter），避免跨进程序列化开销。适用于快速协作场景（代码审查、测试生成）。

特性实现优势
零序列化通信EventEmitter 直接传递 JS 对象比 IPC/WebSocket 快 100x+
共享上下文共享 readFileState + toolPermissionContext避免重复读取文件和权限检查
生命周期绑定父进程退出时自动 abort无孤儿进程风险
隔离性独立 Agent Session + 独立消息历史不污染父会话状态
🔑 一句话总结：tasks/ + migrations/ 是 Claude Code 的 后台任务与配置演进基础设施。7 种任务类型覆盖从本地 Shell 到远程 Agent 的全部场景；DreamTask 通过推测性执行实现"零等待"；InProcessTeammateTask 通过共享内存实现高性能同进程协作；11 个幂等迁移脚本处理模型升级和配置结构变更。

🔄 Migrations — 幂等配置迁移系统 36

36.2 迁移模式

  • 幂等性：每个迁移先检查"是否已迁移"标记（如字段是否已在目标位置），已完成则跳过
  • 原子性：先写目标 → 再删源，写入失败不会丢失原数据
  • 不可逆：迁移没有 rollback——降级版本时旧配置已被清理
  • 日志追踪：每个迁移通过 logEvent() 记录成功/失败事件
  • 容错启动：迁移失败只 logError，不 throw——不阻塞应用启动
