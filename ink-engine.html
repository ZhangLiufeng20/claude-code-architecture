🖼️ Ink 渲染引擎 20

🔬 方法论：本章对 src/ink/（93 文件 / 13,306+ 行）进行源码级深度分析。 核心文件：ink.tsx（1722行）· render-node-to-output.ts（1462行）· selection.ts（917行）· output.ts（797行）· parse-keypress.ts（801行）· dom.ts（485行）· log-update.ts（773行）· styles.ts（771行）。

ink/ 不是简单的 Ink 框架使用——它是一个完全重写的 React → Terminal 渲染引擎。 它包含自定义 DOM 层、Yoga 布局引擎集成、双缓冲 Screen Buffer、DECSTBM 硬件滚动优化、 全屏文本选择/复制、BSU/ESU 原子更新、以及对 20+ 种终端协议的完整解析。

20.1 自定义 DOM 层

dom.ts（485行）定义了一套类似浏览器的 DOM 树，每个节点包含 Yoga 布局节点、样式、dirty 标记和滚动状态。

7 种节点类型

节点类型Yoga 节点职责
ink-root✅根节点，持有 focusManager 和 onRender 回调
ink-box✅Flexbox 容器，支持 overflow/scroll/clip/noSelect/边框
ink-text✅文本容器，带 measureTextNode 自定义测量函数
ink-raw-ansi✅预渲染 ANSI 内容（如 ColorDiff），跳过 squash/wrap/style
ink-virtual-text❌虚拟文本（不占用布局空间）
ink-link❌OSC 8 超链接包装器
ink-progress❌进度条（无布局影响）
#text❌纯文本节点（nodeValue）

markDirty 上浮策略

任何 DOM 变更（setAttribute、setStyle、appendChild、removeChild、setTextNodeValue）都触发 markDirty()：

  1. 抬升 dirty 标记：从当前节点向上遍历祖先链，所有 DOMElement 的 dirty=true
  2. Yoga 标脏：仅在叶节点（ink-text 或 ink-raw-ansi）上调用 yogaNode.markDirty() — 触发重新测量
  3. shallowEqual 优化：setStyle 和 setTextStyles 会先比较新旧值，React 每次渲染创建新对象但值不变时不触发 markDirty

子节点移除清理

removeChildNode 触发 collectRemovedRects()，递归收集被移除子树的缓存矩形区域，存入 pendingClears Map。渲染器在下一帧中读取并清空这些区域，防止幽灵内容残留。绝对定位节点的移除会额外标记 fromAbsolute，触发全局 blit 禁用防止覆盖幽灵。

20.2 渲染管线

ink.tsx（1722行）是整个渲染引擎的核心类，管理帧生命周期、React reconciler 挂载、双缓冲帧交换和终端 I/O。

renderNodeToOutput — blit 优先策略

render-node-to-output.ts（1462行）遍历 DOM 树，对每个节点执行：

  1. blit 检查：!node.dirty && cached.{x,y,w,h} === current && prevScreen → 直接从上一帧复制像素
  2. clear 旧位置：当 positionChanged 时，先清除缓存的旧矩形区域
  3. 零高度抑制：height===0 && siblingSharesY() → 跳过渲染防止同行兄弟节点幽灵叠加
  4. 三种渲染路径：ink-raw-ansi（直接 write）、ink-text（squash+wrap+style）、ink-box（clip+scroll+子节点递归）

DECSTBM 硬件滚动

ScrollBox 的 scrollTop 变化时生成 ScrollHint{top, bottom, delta}，log-update.ts 利用它发出硬件滚动指令（CSI SU/SD）而非重写整个视口。两种 drain 策略根据终端类型自动选择：

终端类型检测方式Drain 策略
xterm.js（VS Code）XTVERSION 探针 + TERM_PROGRAM=vscode ≤5 → 全部（慢点击秒响应），(5,12) → step=2，≥12 → step=3，>30 → snap
原生终端（iTerm2/Ghostty）XTVERSION 非 xterm.js max(4, floor(|pending|*3/4))，capped at innerHeight-1（保证 DECSTBM 触发）

Follow-Scroll 文本选择同步

当 sticky scroll 触发 scrollTop = maxScroll 时，consumeFollowScroll() 返回 {delta, viewportTop, viewportBottom}。ink.tsx 使用它将活动的文本选择偏移 -delta 行，模拟原生终端行为（选择跟随文本向上移动，最终裁剪在视口顶部）。

20.3 屏幕缓冲区与 Diff

output.ts（797行）收集渲染树的写入操作，应用到 Screen Buffer；log-update.ts（773行）将前后帧 diff 生成终端输出。

操作用途关键细节

双缓冲帧交换 + BSU/ESU

Ink 类维护 frontFrame（当前屏幕）和 backFrame（上一帧）。每帧渲染后交换：

  • log-update.render(frontFrame, newFrame) 生成 diff patches
  • optimize() 合并相邻 patches 减少 write 调用
  • 支持 DEC 2026 同步更新协议：BSU（Begin Synchronized Update）和 ESU（End）保证原子性
  • Alt-screen 每帧前缀 CSI H 重置光标（自愈外部 drift），后缀 CSI row;1 H parking 到底部

20.4 全屏交互

全屏模式下，Claude Code 桰接了终端的所有输入/输出，自行实现了文本选择和键盘解析。

文本选择状态机

stateDiagram-v2 [*] --> 无选择 无选择 --> 单击 : mouseDown 单击 --> 拖拽中 : mouseMove 单击 --> 无选择 : mouseUp 拖拽中 --> 已选择 : mouseUp 已选择 --> 无选择 : Esc/clearSelection 无选择 --> 词选择 : doubleClick 无选择 --> 行选择 : tripleClick 词选择 --> 词扩展 : drag 行选择 --> 行扩展 : drag

拽出回弹机制（virtualRow）

当键盘滚动（PgUp/PgDn）将选择端点推出视口时：

  1. virtualAnchorRow/virtualFocusRow 记录真实（未 clamp）位置
  2. 显示位置 clamp 到视口边界，col 重置为边缘（0 或 width-1）
  3. 反向滚动时从 virtual 位置恢复，同时 pop scrolledOffAbove/Below 累积器
  4. 两端都超出同一边缘时自动 clearSelection()（防止 1-cell 幽灵高亮）

scrolledOff 累积器

拖拽滚动时，Screen Buffer 只保存当前视口。离屏文本通过 captureScrolledRows() 在滚动前捕获，存入：

  • scrolledOffAbove[]（顶部离屏，newest 在末尾）+ 并行 scrolledOffAboveSW[]（软换行标记）
  • scrolledOffBelow[]（底部离屏，newest 在头部）+ 并行 scrolledOffBelowSW[]
  • getSelectedText() 将 above + on-screen + below 拼接，使用 joinRows() 将软换行合并回逻辑行

🔑 一句话总结：ink/ 是 Claude Code 的 终端渲染引擎。自定义 DOM 层提供 7 种节点类型 + shallowEqual 脏标记优化；渲染管线通过 blit 跳过干净子树 + DECSTBM 硬件滚动实现 O(变化单元格) 的增量更新；屏幕缓冲区通过 charCache 行缓存 + BSU/ESU 原子更新保证零闪烁；全屏交互实现了完整的文本选择状态机（单击/双击/三击 + 拖拽滚动累积 + virtual-row 回弹）和 6 种终端协议解析器。这 93 个文件共同构建了一个媒体应用级别的终端渲染引擎。
