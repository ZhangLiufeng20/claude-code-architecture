🔬 方法论：本章对 tools/AgentTool/（15 文件 / 500KB+）进行源码级深度分析。 核心文件：AgentTool.tsx（234KB/1398行）· UI.tsx（125KB）· runAgent.ts（36KB/974行）· loadAgentsDir.ts（26KB）· agentToolUtils.ts（23KB）。

AgentTool 是 Claude Code 的 多智能体协调引擎——它让主循环能够将任务委派给子智能体（SubAgent）， 支持同步/异步/多智能体团队/远程执行 4 种模式，以及 worktree 文件隔离、MCP 服务器继承、技能预加载等高级能力。

27.2 多智能体模式

模式实现隔离级别特点
同步 SubAgentrunAgent + wrapWithCwd共享 AbortController阻塞主循环，共享上下文
异步 BackgroundregisterAsyncAgent + void runAsyncAgentLifecycle独立 AbortController用户 ESC 不会中止，任务通知机制

RemoteteleportToRemote() + CCR完全隔离ant-only，远程环境执行
WorktreecreateAgentWorktree()git worktree文件系统隔离，无变更自动清理

关键安全机制

  • 权限继承：Agent 根据 agentDefinition.permissionMode 设置独立权限；父级 bypassPermissions/acceptEdits/auto 始终优先
  • 工具池隔离：workerTools 通过 assembleToolPool(workerPermissionContext) 独立构建，不继承父级限制
  • CLAUDE.md 优化：Explore/Plan 只读 Agent 跳过 claudeMd + gitStatus（节省 ~5-15 Gtok/周）
  • MCP 安全：strictPluginOnlyCustomization 下，用户 Agent 的 frontmatter MCP 被阻断，仅 admin-trusted 通过
  • 递归防护：fork 子 Agent 通过 querySource + 消息扫描双重检查禁止再次 fork
🔑 一句话总结：tools/AgentTool/ 是 Claude Code 的 多智能体协调引擎。AgentTool.tsx（234KB）通过 4 级类型解析（teammate → fork → 指定类型 → 默认）将任务分发到 6 种执行模式（同步/异步/fork/teammate/remote/worktree）；runAgent.ts（36KB）实现完整的 Agent 生命周期——从 MCP 初始化、技能预加载、Hooks 注册到 query() 主循环 + sidechain 转录 + 资源清理；安全设计以权限继承、工具池隔离、CLAUDE.md 优化、MCP 安全门、递归 fork 防护 5 层机制保障多智能体协作的安全性。

27.3 Coordinator 模式 (coordinatorMode.ts)

coordinator/coordinatorMode.ts（370行/19KB）定义了 Claude Code 的 Coordinator 多智能体编排系统——当启用时，主循环从"直接执行工具"切换为"指挥 Worker 执行任务"模式：

模式控制函数

函数职责关键实现
isCoordinatorMode()检测是否启用双条件：feature('COORDINATOR_MODE') 构建标志 + CLAUDE_CODE_COORDINATOR_MODE 环境变量
matchSessionMode()恢复会话模式会话可能存储了 coordinator/normal 模式，恢复时通过修改 env var 切换——环境变量实时读取无缓存
getCoordinatorUserContext()注入 Worker 上下文生成 workerToolsContext 字符串，描述 Worker 可用的工具列表 + MCP Server 列表 + Scratchpad 目录
getCoordinatorSystemPrompt()生成协调员系统提示370行完整的指挥手册——角色定义/工具使用/Worker 管理/任务流程/提示工程

Worker 工具集管理

  • 简单模式 (CLAUDE_CODE_SIMPLE)：Worker 仅获得 Bash + FileRead + FileEdit 三件套
  • 完整模式：从 ASYNC_AGENT_ALLOWED_TOOLS 中过滤掉内部 Worker 工具（TeamCreate/TeamDelete/SendMessage/SyntheticOutput）
  • MCP 继承：连接的 MCP Server 名称会被注入到 Worker 上下文描述中
  • Scratchpad 共享：当 tengu_scratch 特性门开启时，Worker 获得免权限的共享目录路径

Coordinator System Prompt 结构

章节内容
§1 角色定义协调员——指挥 Worker 研究/实现/验证代码变更，同时直接回答简单问题
§2 工具说明AgentTool（创建 Worker）+ SendMessage（续传 Worker）+ TaskStop（停止 Worker）+ subscribe_pr_activity
§3 Worker 管理使用 subagent_type: "worker"；Worker 自治执行；完成后通过 task-notification XML 报告结果
§4 任务流程四阶段：Research → Synthesis → Implementation → Verification；并行是核心优势
§5 提示工程禁止 "based on your findings"（懒委派）；必须合成理解后再下达精确指令；包含文件路径/行号/具体变更
§6 完整示例从用户报告 bug → 并行调查 → 合成发现 → 精确修复指令 → 验证 的全流程演示
🏗️ 架构洞察：Coordinator 模式的 isScratchpadGateEnabled() 函数特意与 utils/permissions/filesystem.ts 中的 isScratchpadEnabled() 重复实现——这是为了避免循环依赖（filesystem → permissions → ... → coordinatorMode）。Scratchpad 路径通过 QueryEngine.ts 的依赖注入（getCoordinatorUserContext 的 scratchpadDir 参数）传入。

🔑 一句话总结：state/ + context/ 是 Claude Code 的 应用状态基础设施。Zustand-风格的 createStore + useSyncExternalStore 提供性能极致的精确订阅；9 个 Context Provider 覆盖通知队列（4级优先级 + fold 合并）、会话统计、浮层/模态框 UI、语音输入等；AppState 以 toolPermissionContext、mcp、agentDefinitions 三大核心字段串联权限、服务、智能体三大子系统。
