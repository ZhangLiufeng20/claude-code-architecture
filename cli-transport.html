🖨️ print.ts — Headless 运行时引擎（5594行） 39

cli/print.ts 是 -p / --print 模式的 完整运行时—— 它不使用 React Ink，而是直接驱动 query() 循环并通过 NDJSON 或纯文本输出结果。

39.1 Headless 架构

组件职责
runHeadless()主入口。初始化沙箱、恢复会话、启动消息循环
StructuredIONDJSON 格式的标准输出——每行一个 JSON 消息
RemoteIO远程会话 I/O（CCR 协议桥接）
消息队列QueuedCommand 队列 + UUID 去重（10,000 上限 LRU）
Idle Timeout空闲超时自动退出（防止孤儿进程）

39.2 UUID 去重机制

每条入站消息都有 UUID，trackReceivedMessageUuid() 维护一个 10,000 条的 LRU 缓存——超出时驱逐最旧的 UUID， 防止断线重连时重复处理已完成的消息。

39.3 Cron 定时触发

当 feature('AGENT_TRIGGERS') 启用时，print.ts 内建了 cron 调度器：按 cron 表达式定时向消息队列注入 <tick> 标签消息，驱动 Kairos 自主执行任务。

39.4 Team 关闭协议

当 headless 会话中存在 Teammate（多 Agent 协作），退出前必须执行 SHUTDOWN_TEAM_PROMPT——要求主 Agent 先关闭所有 Teammate， 等待 isShutdownApproved() 确认后才真正退出，防止工作丢失。

🔑 设计洞察：print.ts 是 Claude Code 的「无头 REPL」—— 它复制了 REPL.tsx 的全部核心逻辑（权限代理、MCP 管理、会话恢复）， 但去掉了所有 React UI 渲染，改用 NDJSON 协议。 这使得 SDK 客户端（TypeScript/Python）可以通过子进程 stdio 与 Claude Code 交互。
