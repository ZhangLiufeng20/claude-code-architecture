💬 命令系统 08

Claude Code 支持 86+ 个斜杠命令，通过 commands.ts（755行）统一注册和分发。命令通过 /name 触发。

▶ Command 接口定义
export type Command = {
  type: "command";
  name: string;
  description: string;
  // 分组标签（如 history, mcp, config）
  argDescription?: string;
  aliases?: string[];
  isEnabled?: boolean;
  isHidden?: boolean;           // 隐藏但可访问
  options?: CommandOption[];     // 子选项
  call: (args, onUpdate, context) => Promise<CommandResult>;
};

核心命令分类

分类命令说明
会话管理/clear, /compact, /resume, /undo清除、压缩、恢复、撤销对话
配置/config, /model, /theme, /verbose运行时配置切换
MCP/mcp, /mcp-servers, /mcp-toolsMCP 服务器管理
记忆/memory, /forgetCLAUDE.md 读写
权限/permissions, /allowed-tools权限规则管理
状态/status, /context, /cost, /tokens会话状态查看
任务/tasks, /background, /kill后台任务管理
调试/debug, /dump-prompts, /doctor问题排查
帮助/help, /tips, /version帮助信息
💡 条件加载：许多命令通过 feature() 门控或 process.env 判断条件加载。例如 /vim 仅在 VIM_MODE 特性开启时可用，/review 仅在 COORDINATOR_MODE 下可用。

🔐 权限系统 09

权限系统是 Claude Code 安全性的基石，它决定了 AI 能否执行某个工具操作。

权限模式（PermissionMode）

模式说明行为
default默认模式读取操作自动允许；写入操作需用户确认
plan规划模式只允许只读操作；仅生成操作计划
bypassPermissionsYOLO 模式跳过所有确认（危险！）
acceptEdits接受编辑模式子Agent 默认模式，自动接受文件编辑

权限规则层级

graph TB A["ToolPermissionRulesBySource"] --> B["enterprisePolicy\n企业策略（最高优先级）"] A --> C["projectConfig\n项目 .claude/settings.json"] A --> D["userConfig\n全局 ~/.claude/settings.json"] A --> E["session\n当前会话临时规则"] A --> F["autoClassifier\nYOLO 安全分类器"] B --> G{"合并后判定"} C --> G D --> G E --> G F --> G G --> H["allow / deny / ask"]

工具执行权限检查链

1 tool.validateInput(input) — 输入有效性校验
↓
2 tool.checkPermissions(input, context) — 工具自身权限逻辑
↓
3 canUseTool() Hook — 全局权限规则匹配
↓
4 preToolUseHooks — Pre-hook 审计
↓
5 若 denied → 用户确认对话框 / 错误返回

📦 状态管理 10

Claude Code 使用自实现的 Store 模式（非 Redux/Zustand），通过 useSyncExternalStore 与 React 集成。

▶ AppState 完整字段
type AppState = {
  // 模型配置
  mainLoopModel: string;
  thinkingMode: ThinkingMode;
  agent?: string;              // 当前 Agent 定义
  agentDefinitions: AgentDefinitions;

  // 权限
  toolPermissionContext: ToolPermissionContext;

  // MCP
  mcp: { clients: MCPClient[]; tools: Tool[]; prompts: MCPPrompt[] };

  // 会话状态
  isMessageLoading: boolean;
  verbose: boolean;
  hasPermissionError: boolean;

  // 推测执行
  speculationState: SpeculationState;

  // Agent 注册
  agentNameRegistry: Map<string, AgentId>;

  // Kairos（助手模式）
  kairosEnabled: boolean;

  // UI 提示
  promptSuggestion: { text: string; promptId: string } | null;
};
🔑 订阅机制：useAppState(selector) 使用 useSyncExternalStore + 选择器模式，只在选中的状态切片变化时触发重渲染。禁止返回整个 state 对象（会导致每次变更都重渲染）。

推测执行（Speculation）

Claude Code 实现了预测性预执行——在用户还在打字时，AI 猜测下一步输入并提前执行：

组件说明
speculationStateAppState 中的推测状态（idle / pending / completed）
speculation.ts基于对话上下文预测用户可能提交的 prompt
DreamTask推测任务类型——后台静默执行预测查询
PromptSuggestionUI 中显示的输入建议（基于推测结果）
💡 零延迟体验：推测结果匹配用户实际输入时可直接复用已生成的响应。预测失败则静默丢弃，用户无感知。

// 4. Network Error → 简单重试（最多5次）
