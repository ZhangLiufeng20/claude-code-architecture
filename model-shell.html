安全漏洞热修补案例

  • -S/-G 参数注入：原代码将 git diff -S 的标志类型标为 'none'。攻击向量：git diff -S -- --output=/tmp/pwned，校验器看到 -S 无参 → 前进 1 token → 遇到 -- 中断 → --output 不再检查 → 任意文件写入。修复：改为 'string'（消耗下一个 token 作为参数）
  • git tag 创建阻断：git tag foo 会在 .git/refs/tags/ 创建文件，违反只读不变量。通过 additionalCommandIsDangerousCallback 检测无 -l/--list 的 positional arg → 阻断
  • git reflog 写操作：git reflog expire/delete 直接写 .git/logs/，通过子命令黑名单阻断
  • --server-option 排除：git ls-remote --server-option= 向远端发送任意数据（网络写入），即使命令本身是"读操作"
  • UNC 路径：containsVulnerableUncPath 防止 \\\\attacker\\share 触发 Windows NTLM 凭据泄漏

🔑 架构洞察：模型层和 Shell 层虽然功能域不同，但共享一个核心设计理念：精确枚举 + 安全降级。 模型层通过 5 级优先级 + Allowlist + 遗留重映射 + 订阅检查确保 AI 永远使用最合适的模型； Shell 层通过 1,893 行的白名单（20+ git 子命令 × 每个命令 10-40 个安全标志）确保命令执行永远安全。 两者都遵循 fail-closed 原则——枚举之外的一切都被拒绝。
