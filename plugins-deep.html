版本化缓存策略

  • 缓存路径：~/.claude/plugins/cache/{marketplace}/{plugin}/{version}/
  • 临时名称：temp_{prefix}_{timestamp}_{random}（prefix 映射安装源）
  • 原子替换：先安装到 tempPath → 读 manifest → 计算 finalName → rename(tempPath, finalPath)
  • 幂等性：如果 finalPath 已存在 → rm -rf 旧版本 → rename；失败时清理 tempPath

21.3 Schema 与安全

schemas.ts（1,682 行）是类型安全的基座，使用 Zod v4 + lazySchema 模式构建了完整的验证体系。

4 层安全防线

来源验证validateOfficialNameSource保留名称只能来自 anthropics/ 官方 GitHub org

🔑 一句话总结：utils/plugins/ 是 Claude Code 的 插件操作系统。pluginLoader 编排 6 种安装源通过临时目录 → 原子 rename 实现幂等缓存；marketplaceManager 用意图层/状态层双层架构 + Seed 目录注册支持企业级分发；schemas.ts 用 Zod v4 构建 4 层名称安全防线（白名单 + 仿冒正则 + 同形文字检测 + 来源组织验证）；validatePlugin 区分运行时宽松/CLI 严格两种校验模式。这 44 个文件共同构建了一个安全、可扩展、企业就绪的插件平台。
